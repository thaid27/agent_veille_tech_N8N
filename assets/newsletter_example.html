#1 SoulX Singer

Description: SoulX Singer generates singing voices from a few seconds of reference audio. It lets users input a reference voice plus reference singing (or a hummed melody) to make that voice sing new lyrics. It's aimed at creators doing cover songs, composers who want a voice to sing their melodies, and anyone wanting flexible singing voice cloning.

Key Features:

  • • Converts a short reference voice sample into singing.
  • • Can use a reference singing clip or a hummed melody plus lyrics.
  • • Supports varied styles (metal, rap, pop examples shown).
  • • Small model size (under 3 GB) so can run on low-end GPUs or CPU.

Particularities:

Released with a GitHub repo and Hugging Face page; runnable locally. Lightweight (under 3 GB) enabling use on consumer hardware.

Comparison:

Not compared directly to other singing models in the video, but emphasized for its flexibility and small size enabling local use.

#2 Just Dub It

Comparison:

Highlighted benefit is integrated dubbing + lip-sync; leverages LTX2 for speed/efficiency.

#4 Seedance 2.0

Description: Seedance 2.0 is ByteDance/seed's next-generation multimodal AI video generation model (part of seed lab) that creates coherent, high-quality short videos from combinations of image, video, audio and text references. It targets creators, indie filmmakers, animators, advertisers and 3D artists who need cinematic, physics-aware, character-consistent video generation and advanced editing-like features. Showcased in early access and leaks with an API launch reported; reviewers position it as a major leap in video generation quality, especially for action/fight choreography, camera direction, character consistency and native audio.

Key Features:

  • • Multimodal omni model supporting four input types: image, video, audio, and text.
  • • Accepts up to 12 reference inputs per generation (up to nine images + three videos in some reports).
  • • Text-to-video generation (including multi-shot/narrative-capable outputs).
  • • Image-to-video and video-to-video generation and restylization; first-frame/last-frame generation and time-loop generation.
  • • Supports multiple input modes: all-round reference, first-frame/last-frame, intelligent multiframe, main-reference (note some modes not always supported for Seedance 2.0).
  • • Multi-shot "director mode" for chaining shots and camera-angle changes within one generation.
  • • Motion and camera tracking to maintain consistent characters and camera moves across shots.
  • • Expression and camera transfer: transfer expressions/emotions and camera motion from a reference video to a new character.
  • • Native audio output and audio input support (3–15 seconds input, 4–15 seconds selectable native output); audio generation and lip-sync, including multilingual speech.
  • • Cinematic effects: slow motion, bullet time, shot variety, automated editing-style decisions (automatic cuts, varied perspectives).
  • • Subvert the Plot feature (change narrative/genre of input footage) and video inpainting (add/replace elements like characters).
  • • Supports many styles: anime, 2D cartoons, 3D Pixar-style, commercials, UGC, music videos, storyboards/manga-to-video conversion, UI animations, and more.
  • • Reroll/alternate-output functionality for multiple variations; supports aspect ratio selection and short clips (~10–15 seconds examples).

Particularities:

Early access and leaks: briefly available via BytePlus playground leak and reported available on ByteDance’s Chinese platform Jimming; general/global rollout and API access reportedly planned (one report mentioned Feb 24 API timeline). Closed-source and paid; not open-source. Omni multimodal design — blends multiple reference modalities into a single generation pipeline and produces native audio rather than stitching separate TTS. Internal editor and inferred generation controls for intensity/temperament; model may make autonomous stylistic/framing choices ("internal opinion"). Some decoherence and quirks remain: occasional teleporting/incorrect character placement, background characters becoming inert, gibberish text in wide shots, small object/physics glitches, and temporal hiccups requiring multiple generations and human editing/compositing to polish. Reported raster/output sizes and cost estimates in leaks (example: outputs at ~1470 × 630 (~720) for wide cinema aspect with estimated cost ~$0.72 per 15-second generation (~4.8¢/sec) in one report). May face upcoming content restrictions (rumored potential clamps on realistic modern people / public personalities and IP).

Comparison:

Presented by reviewers as outperforming previous top video models and as "by far the best model out there right now" in some coverage. Compared favorably to Sora 2 and Cling/Kling (e.g., Kling 3.0): Seedance 2.0 reported to create a stronger sense of space and excel at action/fight choreography; Kling 3.0 noted for vintage looks and wardrobe/period accuracy in a side-by-side. Positioned as a direct challenger to recent state-of-the-art video models (Kling/Cling 3.0 and Sora), and an improvement over Seedance 1.5 with expanded multimodal support and improved native audio.

#5 DeepGen 1.0

It's aimed at illustrators, designers, and creators needing both generation and advanced editing.

Key Features:

  • • Image editing: add elements, change text, replace objects, convert style.
  • • Creative utilities like maze solving, alternate view prediction, and "what happens next".
  • • Released with code and model files (large total size).

Particularities:

#11 GLM5 (by Z AI)

Description: GLM5 is a new open-source, multimodal large language model from Z AI designed for chat, agent workflows, coding, and creative multimodal outputs. It targets creators, developers, enterprises, and teams who need an agent-capable model that can plan multi-step tasks, generate code/interactive web apps, produce images and 3D scenes, analyze documents, and run locally for privacy. It claims top-tier performance on benchmarks while being open-source and comparatively cheap to run via Z AI's API.

Key Features:

  • • Agent capability that plans and executes multi-step workflows autonomously (web search, tool use, sandboxed file generation).
  • • Multimodal: accepts text and file inputs (PDFs, images) and outputs documents, spreadsheets, HTML apps, visualizations, games, and more.
  • • Built-in coding/agentic generation for standalone HTML demos (games, piano roll, stock visualizer, infinite terrain, 3D scenes).
  • • Integrated image generation via Z AI’s image model (GLM Image) inside the chat/agent flow.
  • • Large context window: 200,000 tokens (roughly 150,000 words).
  • • Mixture-of-experts architecture enabling huge model capacity with efficient inference (only a subset of params active).
  • • Reinforcement learning fine-tuning plus a system (Slime) to accelerate RL training.
  • • Available to try free on Z.AI chat, API access with low-cost plans, and open-sourced on Hugging Face / GitHub.

Particularities:

Open-source release (models available on Hugging Face; GitHub provides instructions). Model distribution is large (~1.5 TB), targeted at enterprise/private deployments rather than consumer hardware. Architecture: 744 billion parameters in total as a mixture-of-experts; only ~40 billion active per inference. Context window: 200,000 tokens (same as Claude Opus). Uses deepseek sparse attention to handle long contexts efficiently. Incorporates RL training and the Slime system to speed up RL fine-tuning. Reportedly a low hallucination rate on benchmarks compared to other leading models. Cheap API pricing noted (example: $27 per quarter plan mentioned).

Comparison:

Compared to GLM 4.5: substantially larger (parameter count increased from 355B to 744B). Versus top closed models (e.g., Claude Opus, GPT 5.2, Gemini 3 Pro): GLM5 benchmarks reportedly on par with many top closed models and near the top closed-tier performance in third-party leaderboards for open-source models. Context window: same as Claude Opus (200k tokens) but much shorter than Gemini 3 Pro (1M tokens). Reported to have a far lower hallucination rate than several leading closed models on the cited benchmark. Positioned as significantly cheaper per usage than Gemini, GPT, and Claude according to the source.
